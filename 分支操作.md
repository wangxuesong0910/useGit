# 分支

## 什么是分支？

* 在版本控制中，使用多条线同时推进多个任务

## 分支的好处

* 同时并行推进多个功能的开发，提高开发效率
* 各个分支在开发过程中，如果某一分支开发失败，不会对其他分支有任何影响。失败的分支删除重新开始即可

## 分支操作

* 创建分支：
  * git branch [分支名]

* 查看分支：
  * git branch -v

* 切换分支
  * git checkout [分支名]

* 合并分支
  1. 切换到接受修改的分支（被合并，增加新内容）上
     * git checkout [被合并的分支名]
  2. 执行merge命令
     * git merge [有内容或已修复bug的分支名]
